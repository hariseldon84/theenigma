### Context Guidance

Enigma is a Sovereign-Edge Cognitive Operating System focused on reducing Information Debt. It combines Notion (Nexus memory), Supabase (Vault), and Python orchestration with a web UI. Current product capabilities include:
- Commitment and context capture (manual audio + ambient ingestion)
- Open Commitments feed and Life Query (RAG over personal context)
- Sovereign Brief daily synthesis and delivery
- Thought Memory and Grandma's Closet for deferred ideas
- Dashboard views and Chat Import (WhatsApp/Telegram) with canonical people-context refresh

Core product goals are fast commitment retrieval, low-friction context capture, natural-language recall, thought/idea lifecycle support, and unified personal intelligence views.

### Conceptual Mode Model

- **Flow Mode (Doing / In the stream)**
  - Purpose: execution, “what do I do next?”, in-the-moment capture and query.
  - Tone: Milan Minimalism, low chrome, content cards; orb-centric interactions.
  - Primary objects: Now/Next commitments, live recommendations, Life Query, capture, micro-timelines of activity.

  - Purpose: configure, review, audit, connect systems, and understand org/state over time.
  - Tone: more traditional SaaS shell while preserving Milan Minimalism; navigation, tables, filters.
  - Primary objects: connections, memory explorers, history, imports, recommendation policies, health connectors, org & roles.

Global design principle: **Mode toggle is always available**, Flow remains the default mental home; Control is “where you go to tune the machine and explore the full graph.”

### SCAMPER-Developed Concepts (Deepened)

**[Category #52]**: Recommendation Inbox (Substitute)
_Concept_: Substitute raw alerts with a single recommendation inbox where Enigma proposals are ranked by urgency, confidence, and expected benefit. Every item supports Accept, Modify, Defer, Reject with one-tap rationale capture.
_Novelty_: Replaces notification chaos with a governed decision surface that trains personalization.

**[Category #53]**: Commitment Compiler (Combine)
_Concept_: Combine signals from email, chat, voice, calendar, and docs into a compiled daily execution graph. Enigma groups related obligations into "work packages" instead of isolated tasks.
_Novelty_: Treats commitments like build artifacts with dependency-aware packaging.

**[Category #54]**: Adaptive Confidence Slider (Adapt)
_Concept_: Adapt recommendation aggressiveness per user mode (Focus, Routine, Recovery, Crisis). Confidence thresholds dynamically change how much automation is suggested versus held for manual review.
_Novelty_: Personalizes the balance between autonomy and control in real time.

**[Category #55]**: Due-Date Estimator v2 (Modify)
_Concept_: Modify due-date inference by incorporating collaborator responsiveness, historical task velocity, and meeting density. Enigma produces "due date + confidence band" rather than a single static date.
_Novelty_: Adds probabilistic planning instead of deterministic guesswork.

**[Category #56]**: Recommendation Replay (Put to Other Use)
_Concept_: Reuse historical accept/reject decisions as training examples users can inspect and edit. Teams can optionally share anonymized patterns for better cold-start behavior.
_Novelty_: Turns decision history into an explicit, user-auditable learning asset.

**[Category #57]**: Low-Value Noise Pruner (Eliminate)
_Concept_: Eliminate repetitive low-impact nudges by enforcing a minimum "decision value score" for surfaced recommendations. Below-threshold items are silently summarized.
_Novelty_: Makes silence an engineered product behavior.

**[Category #58]**: User-First Scheduling Override (Reverse)
_Concept_: Reverse default automation: Enigma proposes schedules but never applies changes unless user opts into auto-apply mode per category. Suggested plans can be re-simulated instantly after edits.
_Novelty_: Keeps automation subordinate to user sovereignty while preserving speed.

**[Category #59]**: Vital-State Work Modes (Substitute)
_Concept_: Substitute static priority lists with health-aware work modes (Deep, Collaborative, Admin, Recovery) selected automatically from vitals + workload. Task queue and prompt cadence reconfigure by mode.
_Novelty_: Bridges human state and execution strategy continuously.

**[Category #60]**: Recovery-First Rescheduler (Combine)
_Concept_: Combine sleep debt/HRV proxies with commitment criticality to reschedule cognitively heavy items and pre-draft expectation-setting messages to stakeholders.
_Novelty_: Couples recovery intelligence with social coordination, not just self-optimization.

**[Category #61]**: Burnout Scenario Sandbox (Adapt)
_Concept_: Adapt risk forecasting into a user-facing sandbox: "if I accept this week plan, what is burnout risk?" Users can test alternative schedules before committing.
_Novelty_: Introduces simulation-based personal planning.

**[Category #62]**: Explainability Receipt (Modify)
_Concept_: Modify every Enigma suggestion into a compact receipt: source evidence, policy checks, confidence, and expected upside/downside. Users can expand to full trace.
_Novelty_: Makes trust legible at decision time, not in hidden logs.

**[Category #63]**: Policy Breach Circuit Breaker (Eliminate/Reverse)
_Concept_: Eliminate risky autonomous actions by default under uncertain policy conditions; reverse to "require explicit approval" whenever confidence or compliance checks drop below threshold.
_Novelty_: Adds safety-first fail-closed behavior to agent operations.
